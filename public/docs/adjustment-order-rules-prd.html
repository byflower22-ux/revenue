调整单申请页 - 订单规则模块 PRD

1. 背景与目标

  • 为什么做：调整单申请时，用户需要对关联订单的收入分配规则进行调整（修改比例、新增收入类型、变更团队归属），并将调整后的规则与下方的业务信息/交付信息模块联动，确保调整项的"项目"选项与规则中的实际收入类型保持一致。
  • 要达成什么：实现调整项目选择 → 订单规则调整 → 业务信息/交付信息项目下拉的三方联动，提升调整单申请的准确性和操作效率。

2. 用户故事

作为 财务/运营人员，我希望在创建调整单时能够：

  1. 按基础类型选择调整项目，系统自动显示对应的业务信息或交付信息模块
  2. 开启规则调整后，修改订单收入分配规则的比例和团队归属
  3. 在规则中新增收入类型后，下方模块的"项目"下拉自动包含新增的类型
  4. 不调整规则时，也能选择订单已有的收入类型

3. 功能范围

In Scope

  • 调整项目选项改为4个基础类型
  • 订单规则模块的查看与调整（比例、团队、新增/删除收入类型）
  • 业务信息/交付信息模块的条件显示
  • 项目下拉与规则模块的三方联动
  • 已有收入数据按选中类型过滤

Out of Scope

  • 规则模块的独立管理页面（本期仅在调整单内操作）
  • 规则生效/失效的历史版本管理
  • 规则审批流程

4. 功能详细说明

4.1 调整项目选项

  • 输入：多选下拉，选项为4个基础类型：业务收入、导流收入、业务渠道分成、交付收入
  • 处理逻辑：
    • 选择"业务收入"/"导流收入"/"业务渠道分成"中任一项 → 下方显示"业务信息"模块
    • 选择"交付收入" → 下方显示"交付信息"模块
    • 两者均选中 → 两个模块都显示，业务信息在上，交付信息在下
    • 均未选择 → 两个模块均隐藏
  • 输出：已选基础类型列表，用于后续联动

4.2 订单规则模块

  • 显示条件：选择关联订单后展示，默认为只读查看模式
  • 规则摘要区：展示规则名称、版本、生效状态、收入类型数量、比例合计
  • 开启调整：通过开关切换为编辑模式
    • 可编辑比例（0-100%），实时计算合计是否达到100%
    • 可编辑团队归属
    • 可新增收入类型（选择基础类型后自动编号）
    • 可删除新增的收入类型
    • 可重置为原始规则
  • 平衡校验：比例合计 ≠ 100% 时显示警告，不允许提交

4.3 业务信息/交付信息模块联动

  • 项目下拉数据源：
    • 规则调整开启时：从编辑中的规则数据读取
    • 规则调整关闭时：从订单匹配的原始规则数据读取
  • 业务信息的项目选项：过滤规则中被选中的类型所包含的收入项
  • 交付信息的项目选项：过滤规则中交付收入类型
  • 已有数据过滤：仅显示与选中的基础类型匹配的数据
  • 新增行：用户手动添加的调整项不受过滤影响

4.4 交互流程

选择调整项目（4选多）
    │
    ├─ 包含业务收入/导流收入/业务渠道分成
    │       │
    │       ▼
    │   显示【业务信息】模块
    │   项目下拉 ← 规则模块收入类型（过滤匹配）
    │   已有数据 ← 仅显示选中类型的收入行
    │
    ├─ 包含交付收入
    │       │
    │       ▼
    │   显示【交付信息】模块
    │   项目下拉 ← 规则模块交付收入类型
    │
    └─ 规则模块新增收入类型
            │
            ▼
        项目下拉实时更新（无需刷新）

5. 数据说明

5.1 基础类型与收入项映射

基础类型默认收入项说明
业务收入业务收入1、业务收入2可通过规则模块新增业务收入3、4...
导流收入导流收入单项
业务渠道分成业务渠道分成单项，比例可为负数
交付收入交付收入单项

5.2 规则数据结构

字段类型说明
keystring唯一标识
namestring收入项名称
baseTypestring基础类型
rationumber比例（0-1之间）
teamstring团队归属
isNewboolean是否为本次新增
originalRationumber原始比例（用于对比）
originalTeamstring原始团队（用于对比）

6. 验收标准

  • ☑ 调整项目下拉显示4个基础类型选项
  • ☑ 只选"业务收入"时，仅显示业务信息模块
  • ☑ 只选"交付收入"时，仅显示交付信息模块
  • ☑ 同时选业务类型和交付收入时，两个模块都显示
  • ☑ 未选择调整项目时，两个模块均隐藏
  • ☑ 规则模块默认只读，开启开关后可编辑
  • ☑ 规则模块新增"业务收入3"后，项目下拉自动出现该选项
  • ☑ 规则模块关闭编辑后，项目下拉回退到原始规则
  • ☑ 比例合计 ≠ 100% 时显示警告提示

7. 影响范围

  • 前端：AdjustmentFormPage.jsx（主要改动文件）
  • 数据：mockData.js 中的 ruleVersions、revenueTypeMap
  • 不受影响：调整单列表页、订单核算管理页、订单详情页
