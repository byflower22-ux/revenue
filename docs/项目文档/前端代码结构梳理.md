## 2. 每个页面的功能说明

### 2.1 收入核算页 (`revenue`)

左右分栏的订单主从视图。左侧为订单列表（含筛选和分页），右侧为选中订单的详情面板。

- 支持按订单号、团队、区域、产品类型、员工、买家等维度筛选
- 左侧列表支持 Tab 切换（业务收入/导流收入/交付收入）
- 右侧详情展示：订单摘要、收入汇总统计、收入流水表、规则快照时间线、调整记录
- 支持从详情面板发起「规则调整」和「收入调整」

### 2.2 订单核算管理页 (`order-accounting`)

全宽表格式列表页，顶部统计卡片 + 多行筛选器 + 宽表。

- 4 个统计卡片：订单总数、已确认收入、未确认收入、合计金额
- 4 行筛选器（第3-4行可展开/收起）
- 33+ 列宽表，支持行多选
- 支持批量调整规则、导出 Excel
- 点击「查看详情」跳转订单详情页

### 2.3 订单详情页 (`order-detail`)

左右分栏详情页。左侧为订单基础信息侧栏，右侧为收入确认概览 + 明细 Tab。

- 左侧：订单头信息、核心信息、产品/活动信息、人员信息、规则信息
- 右侧：收入确认进度条、分类型统计卡片、确认率
- Tab 页：收入核算明细（16列表格）、规则快照、调整记录
- 支持发起「规则调整」（弹窗选择目标规则版本）

### 2.4 调整单列表页 (`adjustment`)

标准列表页，含筛选、表格、查看详情和日志弹窗。

- 筛选：调整单号、关联业务单号
- 表格展示所有调整单，支持分页
- 「查看」打开详情弹窗，「日志」打开操作日志时间线
- 「新增」跳转调整单申请页

### 2.5 调整单申请页 (`adjustment-form`)

多区块长表单页，用于新建调整单。

- 基础信息：申请人、ERP账号、标题、调整项目、关联业务单号
- 关联订单信息：选中订单后展示订单表格
- 订单规则：展示匹配规则详情，支持开关「是否调整规则」并编辑比例/团队
- 业务信息：新增调整项目表格（可编辑行）
- 交付信息：交付单列表 + 调整项表格（可编辑行）
- 底部：调整原因、审批人、附件上传、备注
- 弹窗：选择业务单号弹窗（表格分页选择）

### 2.6 演示配置管理页 (`demo-admin`)

演示系统的后台管理，含 3 个 Tab：版本管理、标记点管理、文档管理。支持增删改查、重置默认、导出为代码。

## 9. 校验逻辑清单

### 9.1 批量调整规则校验（订单核算管理页）

**入口：** 点击「批量调整规则」按钮 → 弹窗选择目标规则版本 → 确认调整

**前置校验：**

| 校验项 | 触发时机 | 提示方式 | 说明 |
|---|---|---|---|
| 未勾选订单 | 点击「批量调整规则」按钮 | 按钮置灰 `disabled` | `selectedRowKeys.length === 0` 时按钮不可点击 |
| 未选择目标规则 | 弹窗内点击「确认调整」 | 按钮置灰 `disabled` | `batchTargetRule === null` 时按钮不可点击 |
| 目标规则不存在 | 逐单处理时 | 加入失败列表 | 所有订单标记失败，原因：`目标规则版本 xxx 不存在` |

**逐单校验流程（mockData.js `batchUpdateOrderRules`）：**

```
对每个选中订单 orderNo：
  1. 查找订单 → 不存在则失败
  2. 查找旧规则 → 不存在则失败
  3. 计算收入确认明细 (calculateRevenueBreakdown)
  4. 检查是否有任何已确认收入 (confirmed !== 0)
     ├── 无已确认收入 → 跳过比例校验，直接通过
     └── 有已确认收入 → 逐项比较 5 个比例 key：
         业务收入1 (business1Ratio)
         业务收入2 (business2Ratio)
         导流收入   (trafficRatio)
         渠道分成   (channelRatio)
         交付收入   (deliveryRatio)
         └── 对每个已确认的收入项：旧规则比例 ≠ 新规则比例 → 记录不一致
  5. 有不一致 → 加入失败列表（附具体原因）
  6. 无不一致 → 更新订单的 matchedRuleVersion / matchedRuleId，加入成功列表
```

**失败原因示例：**

| 失败场景 | reason 文案 |
|---|---|
| 订单不存在 | `订单 ORDxxx 不存在` |
| 当前规则版本不存在 | `订单当前规则版本 v1 不存在` |
| 比例不一致（多项） | `业务收入1比例不一致: 当前35%, 目标40%; 导流收入比例不一致: 当前15%, 目标20%` |

**结果展示：** 弹窗分成功/失败表格，支持导出 Excel（含成功明细 + 失败原因）。

### 9.6 收入确认逻辑校验（calculateRevenueBreakdown）

此函数用于判断订单的收入确认状态，影响批量调整规则的校验结果：

```
对每个收入类型 item：
  ├── 订单已取消 → confirmed = 0, pending = 0
  ├── 支付触发型（业务收入/导流收入/渠道分成）：
  │   ├── 已支付 && 已交付 → confirmed = amount × ratio
  │   └── 已支付 && 未交付 → pending = amount × ratio
  └── 交付触发型（交付收入）：
      ├── 已交付 → confirmed = amount × ratio
      └── 已支付 && 未交付 → pending = amount × ratio
```

**关键判定：** `hasAnyConfirmed = breakdown.items.some(item => item.confirmed !== 0)`
- 有已确认收入 → 逐项校验旧/新规则比例是否一致
- 无已确认收入 → 免校验直接通过（订单尚未产生确认收入，规则可自由切换）

### 10.4 调整单申请页（AdjustmentFormPage）

#### 10.4.1 调整项目联动 — 模块显隐（核心交互）

`adjustmentItems` 多选值决定哪些区块可见：

```
adjustmentItems 选中值          → 展示的模块
─────────────────────────────────────────────────
业务收入 / 导流收入 / 业务渠道分成 → ✅ 业务信息模块
交付收入                         → ✅ 交付信息模块
无选中                          → 两个模块都不显示
```

具体逻辑：
```js
showBizModule = selectedAdjustItems.some(item =>
  ['业务收入', '导流收入', '业务渠道分成'].includes(item)
);
showDeliveryModule = selectedAdjustItems.includes('交付收入');
```

#### 10.4.2 关联订单信息区块

| 条件 | 展示 |
|---|---|
| `activeOrder` 有值（已选择至少一个订单） | 显示「关联订单信息」区块 + 订单详情表格 |
| `activeOrder` 无值 | 不显示此区块 |

#### 10.4.3 订单规则区块

| 条件 | 展示 |

| `activeOrder && matchedRule`（有订单且有匹配规则） | 显示「订单规则」区块：规则名称/版本/状态 + 规则表格 |
| `activeOrder && !matchedRule`（有订单但无匹配规则） | **整个区块隐藏** |

#### 10.4.4 规则调整模式切换

`adjustRule` 布尔值控制规则编辑区 UI：

| adjustRule | 规则面板 | 操作按钮 |
|---|---|---|
| `false`（查看模式） | 普通样式，只读展示比例和团队 | 「调整规则」按钮 |

#### 10.4.5 规则表格列模式切换

| 列名 | 查看模式（adjustRule=false） | 编辑模式（adjustRule=true） |
