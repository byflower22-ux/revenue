# 批量调整规则功能设计

## 概述

在订单核算管理列表页新增「批量调整规则」功能，支持勾选多条订单后批量切换至新的规则版本。执行时校验已生成有效收入的比例与新规则是否一致，通过的订单直接修改，失败的提供 CSV 文件下载。

## 交互流程

### 入口

- 订单核算列表表格增加左侧 checkbox 列，支持行选择
- 表格上方工具栏区域新增「批量调整规则」按钮，显示已选数量
- 未勾选时按钮 disabled 或点击后提示「请先选择订单」

### 弹窗（单步）

点击「批量调整规则」后弹出 Modal：

1. **已选订单标签区** — 显示勾选的订单号，支持点击 ✕ 移除
2. **规则版本选择** — 下拉选择已有规则版本（从 `ruleVersions` 中取）
3. **规则详情表格** — 选择规则后下方展示完整规则详情表格：
   - 列：条件序号、订单类型、产品、来源、业务收入1比例、业务收入1归属团队、业务收入2比例、业务收入2归属团队、导流收入比例、导流收入归属团队、渠道分成比例、渠道分成归属团队、交付收入比例、交付收入归属团队
   - 行：每个 condition 一行，不同条件行比例和归属团队不同
   - 横向滚动，纵向滚动（max-height 260px）
4. **操作按钮** — 取消 | 确认调整

### 执行结果

点击「确认调整」后直接执行校验和修改，展示结果页：

1. **汇总** — 成功 X 条，失败 Y 条
2. **失败提示区**（仅失败时显示）— 「下载失败文件」链接
3. **结果表格** — 订单号、调整结果（成功/失败）、说明
4. **关闭按钮**

### 失败文件

CSV 格式，列：订单号、原规则版本、目标规则版本、失败原因

## 校验逻辑

对每条已选订单：

1. 通过 `calculateRevenueBreakdown(order)` 获取各收入类型的 confirmedAmount
2. 判断每种收入类型是否有有效数据（confirmedAmount ≠ 0）
3. 对有有效数据的收入类型，比较当前规则的 ratio 与新规则同类型 ratio 是否一致
4. 所有有有效数据的类型比例都一致 → 校验通过 → 直接修改订单的 `matchedRuleVersion` 和 `matchedRuleId`
5. 任一有有效数据的类型比例不一致 → 校验不通过 → 跳过，记录失败原因
6. 订单未生成任何有效收入（所有类型 confirmedAmount = 0）→ 免校验，直接通过

校验范围：全部 5 种收入类型统一校验（业务收入1、业务收入2、导流收入、业务渠道分成、交付收入）。

## 涉及文件

| 文件 | 改动 |
|------|------|
| `App.jsx` | 订单核算表格增加 `rowSelection`；工具栏增加批量调整按钮；新增批量调整 Modal 组件（含状态管理）；新增结果 Modal |
| `mockData.js` | 新增 `batchUpdateOrderRules(orderNos, newRuleVersion)` 函数，封装校验+修改逻辑 |
| `App.css` | 批量调整弹窗相关样式 |

## 数据流

```
勾选订单 → 点批量调整按钮 → 弹窗选规则 → 确认调整
  → 遍历已选订单
    → calculateRevenueBreakdown 获取已确认收入
    → 比较有有效数据的收入类型比例
    → 通过：updateOrder 修改 matchedRuleVersion/matchedRuleId
    → 不通过：记录到失败列表
  → 展示结果
  → 如有失败：生成 CSV 供下载
```
