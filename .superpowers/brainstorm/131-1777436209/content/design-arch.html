设计方案：整体架构

演示标记层完全独立于业务代码

文件结构
src/
  App.jsx           ← 原始代码（不修改）
  OrderDetailPage.jsx  ← 原始代码（不修改）
  AdjustmentPage.jsx   ← 原始代码（不修改）
  AdjustmentFormPage.jsx ← 原始代码（不修改）
  mockData.js         ← 原始代码（不修改）
  demo/                ← 演示标记层（独立目录）
    DemoProvider.jsx   &span style="color: #999;">← 开关状态管理
    IterationMark.jsx  ← 标记包裹组件（区域级）
    FieldMark.jsx     ← 标记包裹组件（字段级）
    DocDrawer.jsx     ← 侧边抽屉（展示PRD+流程图）
    DemoToggle.jsx    ← 右上角开关按钮
    iterations.js     ← 迭代标记配置数据
    docs/               ← PRD文档和流程图资源
      prd/              ← 需求文档HTML
      flows/             ← 流程图图片
切换机制

演示模式 ON

✓ IterationMark 渲染标记徽章
✓ FieldMark 渲染字段标记
✓ 点击标记可打开抽屉
✓ 右上角显示开关按钮

演示模式 OFF（纯净模式）

○ IterationMark → 仅渲染子组件
○ FieldMark → 仅渲染子组件
○ 无任何额外DOM输出
○ 无开关按钮

交付方式

交付前端时只需从 main.jsx 中移除 DemoProvider 包裹，并删除 src/demo/ 整个目录。原始页面组件代码没有任何改动，完全可直接复用。
